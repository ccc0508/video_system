# 短视频推荐系统 — 产品需求文档 (PRD)

## 1. 项目概述

### 1.1 项目背景

以抖音、快手为代表的移动短视频平台已风靡全球，其核心竞争力之一在于**推荐算法**。本项目旨在实现一个短视频用户观看行为分析和推荐系统，涵盖数据生成、用户行为模拟、推荐、预测和聚类分析等核心功能。

### 1.2 项目定位

本系统是**数据结构课程大作业**，核心目标是通过一个完整的推荐系统项目，展示对**线性与非线性数据结构**的综合运用能力，以及对**大数据量**场景下算法效率的处理。

### 1.3 技术选型概览

| 项目 | 选型 |
|------|------|
| 编程语言 | Python 3.10+ |
| GUI 框架 | PyQt5 / PySide6 |
| 数据存储 | 文件存储（JSON / CSV / 自定义二进制） |
| 可视化 | Matplotlib / PyQtGraph（嵌入 GUI） |

## 2. 功能需求

### F1: 视频信息库构建

| 属性 | 说明 |
|------|------|
| **规模** | ≥ 100,000 条视频记录 |
| **生成方式** | 通过程序模拟生成（结合预定义的标签/类目体系） |
| **视频字段** | `video_id`, `title`, `category`, `tags[]`, `duration`, `publish_time`, `author_id` |

**用户操作**：
- 可在界面查看视频库总量统计
- 可按类目 / 标签 / 关键词搜索视频
- 可触发"重新生成视频库"操作

---

### F2: 用户行为模拟

| 属性 | 说明 |
|------|------|
| **规模** | ≥ 10,000 个模拟用户 |
| **行为类型** | 观看（watch）、点赞（like）、收藏（favorite） |
| **行为特征** | 每用户根据偏好标签概率性选择视频观看，模拟真实分布 |

**用户字段**：`user_id`, `name`, `preference_tags[]`, `register_time`

**行为记录字段**：`user_id`, `video_id`, `action_type`, `timestamp`, `watch_duration`

**用户操作**：
- 可查看用户列表，搜索指定用户
- 可查看某用户的历史行为记录
- 可触发"重新生成用户行为数据"操作

---

### F3: 相似用户分析

| 属性 | 说明 |
|------|------|
| **输入** | 指定一个用户 ID |
| **输出** | 与该用户兴趣最相似的 Top-K 用户列表（含相似度得分） |
| **算法** | 基于协同过滤的用户相似度计算（余弦相似度 / Jaccard 系数） |

**用户操作**：
- 输入用户 ID → 展示相似用户排行表
- 可调节返回的 Top-K 数量
- 以表格+可视化图形展示结果

---

### F4: 视频推荐

| 属性 | 说明 |
|------|------|
| **输入** | 指定一个用户 ID |
| **输出** | 为该用户推荐的 Top-N 视频列表（含推荐理由/得分） |
| **算法** | 基于用户的协同过滤（User-Based CF） |

**推荐逻辑**：找到相似用户 → 汇总相似用户观看过但目标用户未观看的视频 → 按加权得分排序

**用户操作**：
- 输入用户 ID → 展示推荐视频列表
- 每条推荐附带推荐理由（如"与你兴趣相似的用户也在看"）
- 可调节推荐数量 N

---

### F5: 视频热度预测

| 属性 | 说明 |
|------|------|
| **输入** | 指定一个视频 ID |
| **输出** | 该视频未来时间段的预测观看热度曲线 |
| **算法** | 基于历史观看数据的时间序列分析（移动平均 / 线性回归） |

**用户操作**：
- 输入视频 ID → 展示历史热度曲线 + 预测趋势线
- 以折线图可视化
- 显示关键统计：总观看量、日均观看、增长率

---

### F6: 视频聚类

| 属性 | 说明 |
|------|------|
| **方法** | 基于观看用户的重叠度构建视频相似度矩阵 → 聚类 |
| **算法** | K-Means 或层次聚类 |
| **输出** | 视频分组结果，每组展示代表性视频和共同特征 |

**用户操作**：
- 可设置聚类数 K
- 以散点图 / 树状图展示聚类结果
- 点击某个类簇可查看该簇内所有视频

---

### F7: 用户聚类

| 属性 | 说明 |
|------|------|
| **方法** | 基于观看行为向量构建用户特征矩阵 → 聚类 |
| **算法** | K-Means 或层次聚类 |
| **输出** | 用户分组结果，每组展示代表性用户和共同兴趣特征 |

**用户操作**：
- 可设置聚类数 K
- 以散点图 / 树状图展示聚类结果
- 点击某个类簇可查看该簇内所有用户及其共同偏好标签

## 3. 非功能需求

### 3.1 界面要求

- **图形界面 (GUI)**：基于 PyQt5/PySide6，支持鼠标操作
- **中文提示**：所有按钮、标签、提示信息均为中文
- **菜单导航**：主窗口包含菜单栏，每个功能 (F1-F7) 对应独立菜单项或侧边栏入口
- **异常处理**：非法输入（如空输入、不存在的 ID、非数字输入）弹窗提示

### 3.2 数据存储要求

- **文件存储**：所有数据存储在文件中（不使用数据库）
- 视频库 → `videos.json` 或 `videos.csv`
- 用户库 → `users.json`
- 行为日志 → `behaviors.csv`（CSV 格式利于大数据按行读写）
- 聚类/推荐结果 → `results/` 目录下按功能存储

### 3.3 性能要求

| 操作 | 期望响应时间 |
|------|-------------|
| 视频/用户搜索 | < 1 秒 |
| 相似用户计算 (F3) | < 10 秒 |
| 视频推荐 (F4) | < 10 秒 |
| 热度预测 (F5) | < 5 秒 |
| 聚类分析 (F6/F7) | < 30 秒（可含进度条） |
